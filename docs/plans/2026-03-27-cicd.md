# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions workflows for automated testing on every PR, linting, and Docker image build — so no broken code reaches main.

**Architecture:** Three workflows: `test.yml` runs the full pytest suite against a PostgreSQL service container on every push/PR; `lint.yml` runs ruff for fast feedback; `docker.yml` builds and validates the Docker image on pushes to main. All workflows use the same Python 3.12 + PostgreSQL 16 versions as dev.

**Tech Stack:** GitHub Actions, PostgreSQL 16 service container, Python 3.12, ruff, Docker

---

## File Structure

```
.github/
  workflows/
    test.yml               CREATE  — pytest suite with Postgres service container
    lint.yml               CREATE  — ruff linter
    docker.yml             CREATE  — Docker image build validation
pyproject.toml             MODIFY  — add ruff to dev deps + config
```

---

## Key Design Decisions

### PostgreSQL in CI
GitHub Actions supports service containers. We run PostgreSQL 16 as a service, then run Alembic migrations + pytest against it — same as local dev.

### No Temporal in CI
Temporal server is too heavy for CI. Agent tests use mock providers and the session injection pattern — they don't need Temporal. The 193 existing tests all pass without Temporal.

### Ruff over black/isort/flake8
Single tool, fast, covers formatting + linting. Zero config needed beyond pyproject.toml.

### Docker build but no push
We validate the image builds on main. Pushing to a registry (ECR/GHCR) is deferred to the production deployment plan.

```yaml
name: Lint

on:
  push:
    branches: [main, master]
  pull_request:

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Install ruff
        run: pip install ruff

      - name: Run ruff
        run: ruff check src/ tests/
```

### Task 2: Test workflow with PostgreSQL service

**Files:**
- Create: `.github/workflows/test.yml`

- [ ] **Step 1: Create test workflow**

Create `.github/workflows/test.yml`:

```yaml
name: Test

on:
  push:
    branches: [main, master]
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: launchlens
          POSTGRES_PASSWORD: password
          POSTGRES_DB: launchlens_test
        ports:
          - 5433:5432
        options: >-
          --health-cmd "pg_isready -U launchlens"
          --health-interval 5s
          --health-timeout 5s
          --health-retries 5

      postgres-dev:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: launchlens
          POSTGRES_PASSWORD: password
          POSTGRES_DB: launchlens
        ports:
          - 5432:5432
        options: >-
          --health-cmd "pg_isready -U launchlens"
          --health-interval 5s
          --health-timeout 5s
          --health-retries 5

    env:
      DATABASE_URL: postgresql+asyncpg://launchlens:password@localhost:5432/launchlens
      DATABASE_URL_SYNC: postgresql://launchlens:password@localhost:5432/launchlens
      JWT_SECRET: ci-test-secret-minimum-32-characters-long
      REDIS_URL: redis://localhost:6379/0
      USE_MOCK_PROVIDERS: "true"

    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.12"

      - name: Cache pip
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: ${{ runner.os }}-pip-${{ hashFiles('pyproject.toml') }}
          restore-keys: ${{ runner.os }}-pip-

      - name: Install dependencies
        run: pip install -e ".[dev]"

      - name: Run Alembic migrations (dev DB)
        run: alembic upgrade head

      - name: Run Alembic migrations (test DB)
        env:
          DATABASE_URL_SYNC: postgresql://launchlens:password@localhost:5433/launchlens_test
          DATABASE_URL: postgresql+asyncpg://launchlens:password@localhost:5433/launchlens_test
        run: alembic upgrade head

      - name: Run tests
        run: pytest --tb=short -q
```

### Task 3: Docker build workflow + tag

**Files:**
- Create: `.github/workflows/docker.yml`

- [ ] **Step 1: Create Docker build workflow**

Create `.github/workflows/docker.yml`:

```yaml
name: Docker Build

on:
  push:
    branches: [main, master]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build image
        uses: docker/build-push-action@v6
        with:
          context: .
          push: false
          tags: launchlens:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```
